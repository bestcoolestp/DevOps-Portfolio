# 2026-04-22 — AWS Macie

# Core Mental Model

When you see:

```text id="sec1"
Sensitive data inside S3
```

Think:

```text id="sec2"
Amazon Macie
```

---

# 1. What is Amazon Macie?

Amazon Macie is a fully managed data security and privacy service.

Its job is to:

- Discover sensitive data
- Classify sensitive data
- Alert when sensitive data is found

inside Amazon S3.

---

# Core Purpose

```text id="sec3"
Protect sensitive information stored in S3
```

---

# 2. Main Use Case

Macie scans S3 buckets looking for:

- Personally Identifiable Information (PII)
- Financial information
- Credentials
- Sensitive business data

---

# Examples

Macie can detect:

```text id="sec4"
Names
Addresses
Phone numbers
Email addresses
Credit card numbers
Passport numbers
National IDs
```

---

# 3. How Macie Works

```text id="sec5"
S3 Bucket → Macie Scan → Sensitive Data Found → Finding Generated → EventBridge → SNS / Lambda / Automation
```

---

# 4. Detection Methods

Macie uses:

### Machine Learning

Learns patterns of sensitive information.

---

### Pattern Matching

Recognizes known formats such as:

```text id="sec6"
Credit Card Numbers, SSNs, Passport Numbers, Phone Numbers
```

---

# Example

Macie identifies:

```text id="sec7"
123-45-6789
```

as a Social Security Number.

---

# 5. Data Classification

Macie classifies data stored in S3.

Examples:

| Classification | Example |
|------------|----------|
| PII | Name, Email |
| Financial | Credit Card |
| Credentials | Password Files |
| Confidential | Internal Documents |

---

# 6. Findings

When sensitive data is discovered:

Macie generates a:

```text id="sec8"
Finding
```

---

# Finding Examples

- Public S3 bucket contains PII
- Credit card numbers detected
- Large amount of sensitive records discovered

---

# 7. EventBridge Integration

Macie findings automatically integrate with:

```text id="sec9"
Amazon EventBridge
```

---

# Common Workflow

```text id="sec10"
Macie Finding → EventBridge → SNS → Security Team Alert
```

---

# Or

```text id="sec11"
Macie Finding → EventBridge → Lambda → Automatic Remediation
```

---

# Example Remediation

Sensitive data detected in public bucket.

Lambda automatically:

```text id="sec12"
Block Public Access
```
