# 2026-04-17 — AWS Certificate Manager (ACM)

# Core Mental Model

AWS Certificate Manager (ACM) = managed TLS/SSL certificate service for AWS workloads.

👉 ACM handles:

- certificate provisioning
- validation
- deployment
- renewal

without manual certificate management.

---

# 1. Why ACM Exists

Without ACM:

❌ manual certificate creation  
❌ manual renewals  
❌ expiration outages  
❌ operational complexity

ACM solves:

✅ automated HTTPS  
✅ automatic renewal  
✅ centralized certificate management

---

# 2. What TLS/SSL Actually Does

TLS certificates provide:

```text id="acm1"
Encryption In Transit
````

---

# Example

```text id="acm2"
HTTP  → plaintext
HTTPS → encrypted
```

---

# Important Security Insight

TLS protects against:

👤 man-in-the-middle attacks

by encrypting traffic between:

* browser/client
* AWS service

---

# 3. ACM Core Integrations

ACM integrates natively with:

| AWS Service           | Supported |
| --------------------- | --------- |
| ALB                   | ✅         |
| NLB                   | ✅         |
| CloudFront            | ✅         |
| API Gateway           | ✅         |
| Classic Load Balancer | ✅         |

---

# Important Limitation

❌ Public ACM certificates cannot be directly installed on EC2 instances.

---

# Why?

ACM is designed for managed AWS services.

Not direct OS-level certificate management.

---

# 4. Requesting Public Certificates

You request certificates for:

```text id="acm3"
Fully Qualified Domain Names (FQDN)
```

---

# Examples

```text id="acm4"
api.example.com
corp.example.com
```

---

# Wildcard Certificates

Supported.

Example:

```text id="acm5"
*.example.com
```

---

# Benefit

Single certificate protects:

* app.example.com
* api.example.com
* admin.example.com

---

# 5. Validation Methods

# Email Validation

ACM sends validation email to:

* admin@
* administrator@
* webmaster@
* etc.

---

# Drawback

❌ manual
❌ harder to automate

---

# 6. DNS Validation (Preferred)

ACM provides:

```text id="acm6"
CNAME record
```

---

# Workflow

```text id="acm7"
ACM
↓
Add CNAME to DNS
↓
AWS validates ownership
↓
Certificate issued
```

---

# Major Advantage

✅ fully automatable
✅ ideal for DevOps/IaC

---

# Route 53 Integration

If Route 53 hosts DNS:

✅ ACM can create validation records automatically.

---

# 7. Automatic Renewal

Public ACM certificates renew automatically.

---

# Timing

Renewal begins roughly:

```text id="acm8"
60 days before expiration
```

---

# Huge Operational Benefit

Prevents:

❌ certificate expiration outages

---

# 8. Importing Certificates

You may import externally generated certificates.

Examples:

* enterprise CA
* third-party CA
* on-prem certificates

---

# Critical Limitation

❌ imported certificates do NOT auto-renew.

---

# Operational Responsibility

You must:

✅ monitor
✅ rotate
✅ renew manually

# 12. API Gateway Integration

API Gateway supports ACM custom domains.

---

# Endpoint Types Matter

| Endpoint Type  | Certificate Region Requirement |
| -------------- | ------------------------------ |
| Edge-Optimized | us-east-1                      |
| Regional       | same region                    |
| Private        | ACM generally irrelevant       |

---

# 13. Why Edge-Optimized Requires us-east-1

Because edge-optimized APIs use:

```text id="acm13"
CloudFront
```

internally.

CloudFront certificates must exist in:

```text id="acm14"
us-east-1
```

---

# Extremely Common Exam Trap

If:

* CloudFront
* Edge-Optimized API Gateway

➡️ think:

```text id="acm15"
Certificate must be in us-east-1
```
