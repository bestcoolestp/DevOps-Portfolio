# 2026-04-13 — AWS KMS (Key Management Service)

# Core Mental Model

AWS KMS = centralized cryptographic key management service for AWS.

👉 KMS does NOT primarily store data.

It securely manages:

- encryption keys
- access control
- key rotation
- auditing

for AWS encryption operations.

---

# 1. Why KMS Exists

Without centralized key management:

❌ hardcoded secrets  
❌ unmanaged encryption keys  
❌ poor auditing  
❌ inconsistent encryption policies

KMS solves:

✅ centralized encryption governance  
✅ secure key lifecycle management  
✅ integrated AWS encryption  
✅ auditable cryptographic operations

---

# 2. KMS Core Responsibilities

KMS handles:

- key creation
- encryption/decryption operations
- key rotation
- access policies
- audit logging

---

# 3. Deep AWS Integration

KMS integrates natively with:

| Service | Integration |
|---|---|
| S3 | SSE-KMS |
| EBS | volume encryption |
| RDS | DB encryption |
| DynamoDB | table encryption |
| Lambda | env var encryption |
| SSM Parameter Store | secure strings |

---

# 4. Symmetric vs Asymmetric Keys

## 🔐 Symmetric Keys (AES-256)

Same key used for:

- encryption
- decryption

---

# Most Common KMS Usage

Examples:

- S3 SSE-KMS
- EBS encryption
- RDS encryption

---

# Characteristics

✅ fast  
✅ efficient  
✅ AWS-native standard

---

# 5. 🔑 Asymmetric Keys (RSA / ECC)

Uses:

- public key
- private key pair

---

# Workflow

Public key:

```text
Encrypt
````

Private key:

```text
Decrypt
```

---

# Common Use Cases

* external encryption
* digital signatures
* hybrid cryptography

---

# 6. Key Categories

## AWS Owned Keys

Managed entirely by AWS.

Examples:

* SSE-S3 internal encryption

---

# Characteristics

✅ free
✅ invisible to customers

---

# 7. AWS Managed Keys

Created automatically per AWS service.

Examples:

```text id="kms1"
aws/rds
aws/ebs
aws/s3
```

---

# Characteristics

✅ free
✅ automatic yearly rotation
✅ easier management

---

# 8. Customer Managed Keys (CMKs)

Created and controlled by customers.

---

# Characteristics

✅ custom policies
✅ custom rotation
✅ cross-account access
✅ imported key material possible

💵 typically:

```text id="kms2"
$1/month per key
```

---

# DevOps/SRE Perspective

CMKs are preferred for:

* enterprise governance
* compliance
* separation of duties
* regulated workloads

---

# 9. Key Rotation

## AWS Managed Keys

Automatic yearly rotation.

---

## Customer Managed Keys

Supports:

* automatic rotation
* on-demand rotation

---

## Imported Keys

❌ manual rotation only

---

# Why Rotation Matters

Limits impact of:

* compromised keys
* long-lived cryptographic exposure

---

# 10. Regional Scope (Critical)

KMS keys are:

```text
Region-specific
```

---

# Important Consequence

Cross-region copy operations require:

* re-encryption
* destination-region key

---

# Example

Encrypted EBS snapshot:

```text id="kms3"
us-east-1
```

copied to:

```text id="kms4"
ap-northeast-2
```

requires new KMS encryption.

---

# 11. Key Policies

---

# Default Policy

Allows account usage if IAM permits.

---

# Custom Policy

Can explicitly allow:

* users
* roles
* external accounts

---

# Enterprise Use Case

Cross-account encrypted S3 sharing.

# 13. Common Encryption Workflow

Application
↓
KMS generates data key
↓
Data encrypted locally
↓
Encrypted data stored in S3/EBS/RDS

---

# Important Insight

KMS often protects:

👉 encryption keys
NOT the bulk data directly.

---

# 14. Envelope Encryption (Critical Concept)

Most AWS services use:

```text
Envelope Encryption
```

---

# Process

1. KMS generates data key
2. Data encrypted with data key
3. Data key encrypted with KMS master key
4. Encrypted data + encrypted key stored together

---

# Benefit

✅ scalable
✅ efficient
✅ secure

---
