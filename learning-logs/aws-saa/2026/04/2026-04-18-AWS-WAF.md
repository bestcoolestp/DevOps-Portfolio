# 2026-04-18 — AWS WAF (Web Application Firewall)

# Core Mental Model

AWS WAF = Layer 7 security firewall for HTTP/HTTPS applications.

👉 WAF inspects:

- HTTP requests
- headers
- body
- query strings
- URI paths

and blocks malicious traffic before it reaches your application.

---

# 1. Why AWS WAF Exists

Without WAF:

❌ SQL injection attacks  
❌ XSS attacks  
❌ bot abuse  
❌ Layer 7 DDoS floods  
❌ malicious payloads

can directly hit applications.

---

# WAF Goal

```text id="waf1"
Filter malicious HTTP traffic before application processing
```

---

# 2. OSI Layer Understanding (Critical)

# AWS WAF

Operates at:

```text id="waf2"
Layer 7 (Application Layer)
```

---

# Protects

* HTTP
* HTTPS
* GraphQL APIs

---

# Does NOT Protect

```text id="waf3"
Layer 4 TCP/UDP traffic
```

---

# Exam Trap

WAF does NOT attach to:

```text id="waf4"
Network Load Balancer (NLB)
```

because NLB operates at Layer 4.

---

# 4. Web ACL (Web Access Control List)

Core WAF object:

```text id="waf5"
Web ACL
```

---

# Purpose

Contains security filtering rules.

---

# Workflow

```text id="waf6"
Incoming Request
↓
WAF Web ACL
↓
Allow / Block / Count
↓
Application
```

---

# 5. Rule Types

# IP Set Rules

Allow/block IP addresses.

Supports:

```text id="waf7"
up to 10,000 IPs
```

per IP set.

---

# Example Use Cases

* block attackers
* allow corporate IPs
* country restrictions

---

# 6. HTTP Inspection Rules

WAF can inspect:

* headers
* cookies
* body
* URI path
* query strings

---

# Example

```text id="waf8"
Block requests containing:
UNION SELECT
```

---

# Why?

Detect SQL injection attempts.

---

# 7. Size Constraint Rules

Restrict request size.

Example:

```text id="waf9"
Reject requests > 2 MB
```

---

# Use Cases

✅ abuse prevention
✅ upload protection
✅ payload filtering

---

# 8. Geo Match Rules

Block or allow traffic by country.

---

# Example

```text id="waf10"
Allow only: US, CA, KR
```

---

# Common Use Cases

* licensing restrictions
* compliance
* regional abuse mitigation

---

# 9. Rate-Based Rules (Very Important)

Protect against:

* brute-force attacks
* scraping
* Layer 7 DDoS

---

# Example

```text id="waf11"
Block IPs sending > 1000 requests in 5 minutes
```

---

# Major Advantage

Automatic temporary mitigation against abusive clients.

---

# 10. Rule Groups

Reusable collections of rules.

---

# Example

```text id="waf12"
OWASP Top 10 Rule Group
```

---

# Benefit

Reusable enterprise security standards.

---
