# 2026-04-23 — CIDR (Classless Inter-Domain Routing)

# Core Mental Model

CIDR tells us:

```text id="cidr1"
How many IP addresses belong to a network
```

and

```text id="cidr2"
Which part is the network
Which part is the host
```

---

# 1. What is CIDR?

CIDR stands for:

```text id="cidr3"
Classless Inter-Domain Routing
```

It is the modern way of allocating IPv4 addresses.

Format:

```text id="cidr4"
IP Address / Prefix
```

Examples:

```text id="cidr5"
10.0.0.0/16

192.168.1.0/24

172.31.0.0/20
```

---

# 2. Understanding the Prefix

The number after "/" indicates:

```text id="cidr6"
How many bits are fixed for the network
```

IPv4 contains:

```text id="cidr7"
32 bits total
```

---

# Example

```text id="cidr8"
192.168.1.0/24
```

means:

```text id="cidr9"
24 bits = network

8 bits = hosts
```

---

|--------|-------------|

# 5. Special CIDRs

## /32

Represents:

```text id="cidr16"
Exactly one IP
```

Example:

```text id="cidr17"
134.56.78.123/32
```

Useful in:

- Security Groups
- NACL rules
- Allowing one administrator IP

---

## /0

Represents:

```text id="cidr18"
All IPv4 Addresses
```

Example:

```text id="cidr19"
0.0.0.0/0
```

Meaning:

```text id="cidr20"
Anywhere on the Internet
```

---

# AWS Exam Warning

If Security Group contains:

```text id="cidr21"
0.0.0.0/0
```

for SSH:

```text id="cidr22"
Port 22 open to entire Internet
```

Often considered a security risk.

---

# 9. AWS Reserved IP Addresses

AWS reserves 5 IPs in every subnet.

Example:

```text id="cidr43"
10.0.1.0/24
```

256 total addresses.

Usable:

```text id="cidr44"
251
```

Reserved:

| Address | Purpose |
|-----------|---------|
| .0 | Network |
| .1 | VPC Router |
| .2 | DNS |
| .3 | Future Use |
| .255 | Broadcast Reserved |

---

# 10. CIDR Expansion

Larger network:

```text id="cidr48"
/16
```

Smaller network:

```text id="cidr49"
/24
```

Rule:

```text id="cidr50"
Smaller prefix
=
More IPs
```

Examples:

```text id="cidr51"
/8  > /16 > /24 > /32

Largest → Smallest
```

# Final Memory Anchor

> CIDR defines the size of a network. Smaller prefixes (/8, /16) provide more IP addresses, while larger prefixes (/24, /32) provide fewer IP addresses. In AWS, every subnet reserves 5 IP addresses automatically.
