# 2026-05-14 — AWS Database Migration Service (DMS)

# What is AWS DMS?

AWS DMS means:

```text
AWS Database Migration Service
```

Purpose:

```text
Move databases from On-Premises → AWS
```

---

Key benefit:

```text
Source database stays online.
```

Users can continue using the application during migration.

---

# Core Idea

DMS performs:

```text
Full Load + CDC
```

---

# CDC

CDC means:

```text
Change Data Capture
```

It captures:

```text
INSERT, UPDATE, DELETE
```

changes continuously.

---

Flow:

```text
Source DB → Initial Full Copy → Continuous Sync
```

---

# DMS Architecture

```text
Source Database → DMS Replication Instance → Target Database
```

---

The DMS replication instance is:

```text
An EC2 instance
```

managed by AWS.

---

# Migration Types

There are two types.

---

# 1. Homogeneous Migration

Same database engine.

Examples:

```text
Oracle → Oracle
```

---

```text
PostgreSQL → PostgreSQL
```

---

```text
MySQL → RDS MySQL
```

---

Advantages:

```text
Easy, Fast, & No schema conversion
```

---

# 2. Heterogeneous Migration

Different database engines.

Examples:

```text
Oracle → PostgreSQL
```

---

```text
SQL Server → Aurora MySQL
```

---

```text
Oracle → Redshift
```

---

Requires:

```text
Schema Conversion Tool
```

---

# AWS SCT

SCT means:

```text
Schema Conversion Tool
```

Purpose:

```text
Convert database schema.
```

---

Converts:

```text
Tables, Views, Indexes, Stored Procedures, & Functions
```

---

# SCT Rule

If:

```text
Source Engine = Target Engine
```

↓

```text
No SCT Needed
```

---

If:

```text
Source Engine ≠ Target Engine
```

↓

```text
Use SCT
```

---

# Example

No SCT:

```text
PostgreSQL → RDS PostgreSQL
```

---

SCT Required:

```text
Oracle → Aurora PostgreSQL
```

---

Source:

```text
On-Prem Oracle
```

Target:

```text
RDS MySQL
```

---

Step 1:

```text
SCT → Convert schema
```

---

Step 2:

```text
Create DMS Replication Instance
```

---

Step 3:

```text
Full Load
```

---

Step 4:

```text
CDC
```

---

Final architecture:

```text
Oracle → SCT → DMS → RDS MySQL
```

---

# Multi-AZ DMS

DMS supports:

```text
Multi-AZ
```

---

Architecture:

```text
Primary ↔ Standby
```

---

Benefits:

```text
AZ failure protection

Redundancy

Lower latency spikes

Better availability
```

---

# Migration Workflow

```text
Source DB → Schema Conversion → Full Load → CDC → Target DB
```

---

# DMS Does NOT Do

DMS:

```text
Migrates data
```

---

DMS does NOT:

```text
Convert schemas
```

That is SCT's job.

---

# DMS + SCT Relationship

```text
SCT = Schema
```

---

```text
DMS = Data
```

---

Memory phrase:

```text
SCT = Structure

DMS = Data
```

---

# Final Memory Anchor

> AWS DMS migrates data while keeping the source database online. Use DMS alone for homogeneous migrations and add SCT when migrating between different database engines. DMS performs a full load first and then uses CDC to continuously synchronize changes. Multi-AZ deployments provide resilience during migration.
