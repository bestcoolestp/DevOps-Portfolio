# 2026-05-08 — IPv6 for AWS VPC

# Why Was IPv6 Created?

IPv4 was created in the early days of the internet.

People thought:

```text
4.3 Billion IP Addresses
```

would be enough forever.

---

But now we have:

- Smartphones
- Laptops
- IoT devices
- Servers
- Containers
- Cloud resources

---

IPv4 is running out.

---

# IPv4 Address Space

IPv4 uses:

```text
32 bits
```

---

Total addresses:

```text
2^32
```

=

```text
4,294,967,296
```

≈

```text
4.3 Billion
```

---

Example:

```text
192.168.1.10
```

---

# IPv6 Introduction

IPv6 is the successor to IPv4.

---

IPv6 uses:

```text
128 bits
```

instead of:

```text
32 bits
```

---

Total addresses:

```text
2^128
```

=

```text
340 undecillion
```

≈

```text
3.4 × 10^38
```

addresses.

---

For practical purposes:

```text
Unlimited
```

---

# IPv6 Format

IPv4:

```text
192.168.1.10
```

---

IPv6:

```text
2001:0db8:85a3:0000:0000:8a2e:0370:7334
```

---

Structure:

```text
x:x:x:x:x:x:x:x
```

---

8 groups.

Each group:

```text
0000 → ffff
```

---

Uses:

```text
Hexadecimal
```

---

# Shortened IPv6

Long form:

```text
2001:0db8:0000:0000:0000:0000:abcd:1234
```

---

Can become:

```text
2001:db8::abcd:1234
```

---

Exam tip:

You only need to:

```text
Recognize IPv6
```

not manually calculate it.

---

# IPv4 vs IPv6

| Feature | IPv4 | IPv6 |
|---------|------|------|
| Size | 32 bits | 128 bits |
| Address Count | 4.3 Billion | 3.4 × 10^38 |
| Example | 192.168.1.10 | 2001:db8::abcd |
| Format | Decimal | Hexadecimal |
| Address Exhaustion | Yes | Practically No |

# IPv4 Exhaustion Scenario

Common exam question.

---

Error:

```text
Unable to launch EC2 instance
```

---

Reason:

```text
No IPv4 addresses available
```

---

NOT:

```text
IPv6 exhaustion
```

---

Because:

```text
IPv6 never realistically runs out
```

---

# How To Fix IPv4 Exhaustion

Add another IPv4 CIDR block.

---

Example:

Current VPC:

```text
10.0.0.0/16
```

---

Add:

```text
10.1.0.0/16
```

---

Now the VPC has more addresses.

---

# Do NOT Solve It With IPv6

Exam trap.

---

Question:

```text
EC2 launch fails because IPv4 addresses are exhausted.
```

---

Wrong answer:

```text
Enable IPv6
```

---

Correct answer:

```text
Add another IPv4 CIDR
```

---

Because:

```text
AWS still requires IPv4
```

---

# Final Memory Anchor

> AWS VPC supports dual-stack networking, where IPv4 remains mandatory and IPv6 is optional. IPv6 provides virtually unlimited addresses (3.4 × 10^38), and every AWS IPv6 address is public and internet-routable. If EC2 launches fail because IPv4 addresses are exhausted, the solution is to add another IPv4 CIDR block, not to enable IPv6.
