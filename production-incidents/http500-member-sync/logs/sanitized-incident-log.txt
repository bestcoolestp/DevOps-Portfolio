# **국립대학교(*NU) 연구실안전정보센터 회원정보 동기화 오류 분석 보고서

## 발생 일시

2026-**-**

## 시스템 환경

* ASP.NET MVC (.NET Framework)
* *NU 학사정보 API 연동
* MySQL
* VS2012
* 운영 서버 배포 환경

## 증상

회원정보 동기화 실행 시 다음 오류 발생.

```text
Newtonsoft.Json.Linq.JToken.get_Item(Object key)

위치:
Safety_Biz.Member.MemberMngBiz.SetMemberOracleUpdateJson(...)
```

사용자 화면에는 회원동기화 실패 팝업이 출력됨.

## 실제 원인

학생 API의 일부 페이지 요청 시 *NU API 서버가

```json
status = 500
data = ""
```

응답을 반환함.

그러나 기존 코드에서는 응답 상태를 검증하지 않고 바로 실행.

```csharp
jObject["data"]["students"]
```

이때

```csharp
jObject["data"]
```

는 JObject가 아니라

```csharp
JValue("")
```

형태.

따라서

```text
Cannot access child value on Newtonsoft.Json.Linq.JValue
```

예외 발생.

## 수정 내용

학생 API 응답 검증 로직 추가.

```csharp
if (jObject["status"] == null ||
    jObject["status"].ToString() != "200")
{
    LogWrite("STUDENT API ERROR PAGE = " + i);
    LogWrite(jObject.ToString());

if (!(jObject["data"] is JObject))
{
    LogWrite("STUDENT DATA INVALID PAGE = " + i);
    LogWrite(jObject.ToString());
    continue;
}

if (jObject["data"]["students"] == null)
{
    LogWrite("students is null PAGE = " + i);
```

## 결론

본 장애는 DB 문제, MySQL 문제, 서버 배포 문제, 인증키 문제 때문이 아니라 *NU 학사 API의 비정상 응답 처리 미흡으로 발생하였다.

*NU API가 일부 요청에 대해

```json
{
  "status":"500",
  "data":""
}
```

를 반환할 수 있으므로 모든 API 응답에 대해 상태 검증 후 처리하도록 수정하였다.

현재 회원정보 동기화는 정상 동작한다.
